Для проверки необходимо поднять дамп бд(механизм каскадного удаления книг заложен в ней), 
поднять домен somedomain.com(чтобы не переделывать url)

Api реализованно по задумке что каждый класс работает с одной таблицей в бд, всего 4 основных метода,
но каждый из них внутри может иметь несколько реализаций в зависимости от get параметров.
Таким образом делая запрос put мы подключаемся на изменение информации и можем сделать различную реализацию
исходя из переданных параметров.
Например если передали несколько значений - изменить несколько, одно - изменить одно и.т.п.

--Файлы необходимые для работы
MainApi.php - абстрактный класс для api
Author.php - класс для работы с таблицей авторы
Books.php - класс для работы с таблицей книги
config.php - конфиг
server.php - точка входа
library.sql - дамп бд
.htaccess - необходимые настройки для web сервера


--Пример механизма для отправки запроса и получения ответа
curl.php - функция для отправки запроса
client.php - Примеры запросов
